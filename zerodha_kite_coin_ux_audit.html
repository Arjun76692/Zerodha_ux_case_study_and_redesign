The core friction
One demat account, two separate apps, two broken mental models
A Zerodha user has a single demat account — one identity, one PAN, one broker. Yet to invest in the two most common asset classes, they must operate two completely separate apps: Kite for stocks/ETFs and Coin for mutual funds. Every other major broker in India — Angel One, Groww, Upstox — solved this with a unified experience years ago.

User story: The two-app tax
Rohan is a 28-year-old software engineer who invests monthly. He has ₹5,000 to invest — ₹3,000 into HDFC Bank shares and ₹2,000 into a Parag Parikh Flexi Cap SIP. On Angel One, this is one session, one search bar, two transactions in under 3 minutes. On Zerodha, he must open Kite for the stock, find the right price, place the order — then switch to Coin, log context again in his head, search the fund, set the SIP, invest. Two apps, two separate portfolio screens. He never sees his true net wealth in one place.
Kite: stocks only Coin: MF only No unified portfolio view No cross-product search
What Angel One does right (reference point)
Angel One offers a single home screen with tabs for Stocks, F&O, Mutual Funds, Loans, and FD. The global search bar returns results from all asset classes. Portfolio shows holdings across equities and MFs in one scrollable view. Day's P&L is calculated across the whole account. The invest flow never requires an app switch.

H1 — Visibility of system status
Kite Sev 2
Finding
MF order status lives in Coin, not Kite
After placing a stock order in Kite, the user has no idea a pending MF order (₹1,000 in Parag Parikh) exists. Two separate order screens across two apps prevent a unified status view.
Finding
Processing state with no ETA
Coin shows "Processing" on pending orders but gives no estimated settlement timeline. Users are left uncertain whether their investment has gone through.
H2 — Match between system and real world
Coin Sev 3
Critical finding
Mental model mismatch: same account, different world
Users think of their "Zerodha portfolio" as one thing. The system splits it across two apps with no visual or navigational connection. This breaks the user's mental model of their wealth.
Finding
Coin's discover section feels orphaned
Coin's "Explore" screen (Equity, Debt, Hybrid categories) has a completely different design language from Kite's watchlist, despite being the same brand. Users must re-learn navigation patterns.
H4 — Consistency and standards
Kite Coin Sev 3
Critical finding
Duplicate search — entirely separate systems
Kite has a watchlist search for stocks (34/250 items tracked). Coin has a separate "Search 3000+ mutual funds" bar. There is no cross-search — a user cannot type "HDFC" and see both HDFC Bank stock and HDFC Flexi Cap Fund.
Critical finding
Portfolio tab means two different things
Kite's Portfolio tab shows equity holdings with P&L. Coin's Portfolio shows MF holdings. Neither shows total wealth. Users must mentally add up values across apps to know their net position.
H6 — Recognition over recall
Kite Sev 2
Finding
Kite can search MF names but not invest from them
A user who types "Parag Parikh" in Kite will find the fund. Tapping it prompts them to open Coin. This creates a false start — recognition of the fund fails to complete the task, forcing recall of a second app.
Positive finding
Kite's buy flow is well-designed
The "slide to buy" mechanic with OTP confirmation is intentional friction — it prevents accidental trades. This is a strong pattern worth preserving in any redesign.
H8 — Aesthetic and minimalist design
Coin Sev 1
Finding
Coin's SIPs tab hides active investment status
The SIPs tab shows 1 active SIP but users must navigate into it to confirm it's active. The home state doesn't surface upcoming SIP deduction dates or recent processing confirmations.
Positive finding
Coin's dark UI is clean and readable
Coin's dark-mode-first design with clear hierarchy (fund name → amount → status) is well-executed within its isolated context. The visual design quality is high.

Angel One Unified
What works well Global search, unified portfolio, single home screen with all asset tabs. MF screen shows trending SIPs, fund explorer, and Gold ETF promotions. The "Discover" tab surfaces investments contextually.
Where it breaks Charting is less powerful than Kite. F&O tooling is adequate but not professional-grade. The loan/FD tabs feel like cross-sell noise. Home screen feels busy — too many promotions.
Groww Unified
What works well Best consumer-grade design in Indian fintech. Unified home, clean search, excellent fund comparisons. SIP management is intuitive. Mutual fund discovery (curated lists, peer comparison) is the strongest in the market.
Where it breaks Stock charting is severely limited for active traders. Groww sacrifices depth for breadth. Power users (scalpers, options traders) will always prefer Kite. The unified approach hurts expert tooling.
The design trade-off Zerodha must navigate
Zerodha's core competitive moat is Kite's professional-grade trading tools — charting, F&O depth, order types. Groww proved that unifying apps often means dumbing down for the median user. The redesign must add the unified MF layer without degrading Kite's power tools. The right model: a single app where the MF tab extends Kite's existing portfolio screen — not a full rebrand.
Focused redesign scope
Unified portfolio view + "Invest Now" cross-product flow
Rather than redesigning all of Kite, the proposal is surgical: integrate Coin's MF functionality as a tab inside Kite's Portfolio screen, add a Mutual Funds section to Kite's Discover/search flow, and surface a unified wealth summary at the top of the Portfolio screen. Kite's trading tools are unchanged.

User stories
US-01 — Active investor
As a Zerodha user who invests in both stocks and mutual funds, I want to view my entire portfolio — equities and MFs — in a single screen, so I can understand my true net wealth without switching apps.
Acceptance criteria: Portfolio Overview tab shows combined current value, overall P&L, and asset allocation breakdown. Data refreshes on screen load. MF values use latest available NAV.
US-02 — Casual SIP investor
As a Zerodha user who has set up a monthly SIP in Coin, I want to see my SIP status, next deduction date, and accumulated MF holdings inside Kite, so I don't need to switch to Coin just to check my SIP is active.
Acceptance criteria: Portfolio → Mutual Funds tab lists all SIPs with status, amount, and next deduction date. Users can pause or modify SIPs from this screen.
US-03 — New investor
As a new Zerodha user exploring investment options, I want a single search bar that shows both stocks and mutual funds in results, so I can compare options without knowing in advance which app handles which asset type.
Acceptance criteria: Global search returns grouped results (Stocks / Mutual Funds sections). Result tap navigates to instrument detail; "Invest" CTA available on both. No redirect to external app.
US-04 — Power trader
As an active equity trader who uses Kite for F&O, I want the unified MF features to be additive and not change my existing Kite workflows, so my trading efficiency is unaffected.
Acceptance criteria: Watchlist, chart, order placement, and F&O screens are identical to current Kite. MF features accessible only via Portfolio tab or search. No UI changes to core trading flow.
US-05 — Tax-aware investor
As an investor approaching financial year-end, I want to see my equity STCG/LTCG and MF capital gains in one place, so I can make tax-aware decisions without logging into Console separately.
Acceptance criteria: Portfolio Overview includes a Tax Summary card showing estimated equity and MF capital gains for the current FY. Links to Console for detailed reports.
